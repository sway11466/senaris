# 戦闘演出シーン

2ユニットの戦闘を見せる演出画面の設計。戦闘の解決（損害計算）は [../gdd/combat.md](../gdd/combat.md)／domain 側で確定済みで、本シーンはその結果を表示するだけ（数値を再計算しない）。使う画像スロットは [../art/overview.md](../art/overview.md)、立ち絵の作画方針は [../art/units.md](../art/units.md) §3.3。

盤の見た目は3Dハイブリッド（[../adr/ADR-0003-board-3d-hybrid.md](../adr/ADR-0003-board-3d-hybrid.md)）。演出も同じ3/4俯瞰に寄せ、地形の上に隊列が並ぶ地続き感を出す。

## 配置

- 左右は陣営で固定：プレイヤー左・敵右（攻撃側/防御側で入れ替えない）。立ち絵は向きを陣営に焼き込む（プレイヤー＝右向き／敵＝左向き）ので反転はしない。
- 背景は守り手（防御側）のタイル地形で1枚選ぶ。地形ID→`combat_bg` を autowire（全ユニット共通・ユニットスロットとは別系統）。攻撃側の地形は背景に使わない。

## 兵数の表示（隊列スロット）

- 各陣営の立ち絵（`combat`）を、その時点の兵数（1〜8・[../gdd/combat.md](../gdd/combat.md)）ぶん複製して並べる。同じ1枚を使い回し、スロットの表示ON/OFFで数を合わせる。
- 兵数1〜8の並び位置は3/4俯瞰の隊列テンプレとして事前定義する（前列→後列、重なり時の前後関係を含む）。兵数は戦闘ルールで上限8＝オーバーフロー処理は不要。
- 特別ユニット：`combat_hero` を英雄スロットに1体固定表示し、残り（兵数−1）を base の `combat`（従者）で埋める。英雄が兵数の先頭を占める。
- 立ち絵が未用意のスキンはプレースホルダ（ベタ塗り＋フルネーム）。主要ユニットから順次差し込む（autowire＝置くだけ切替）。

## 動き・エフェクト

- キャラのフレームアニメはしない（立ち絵は静止）。動きはシーン側で付ける：
  - 命中の瞬間に画面シェイク。
  - 被弾側の隊列にフラッシュ。
  - 攻撃側の `combat_effect`（ユニットごと1枚）を相手の隊列上に重ねる（斬撃＝被弾側／魔法＝着弾側）。
- 損害は攻守同時（相討ちあり・[../gdd/combat.md](../gdd/combat.md)）。数値は「敵の失う兵数／自分の失う兵数」を表示。

## フロー

1. 戦闘確定（domain が損害を算出）→ 本シーンへ結果を渡す。
2. 開幕：背景＋両隊列（戦闘前の兵数）を表示。
3. 演出：シェイク＋フラッシュ＋エフェクト、損害数の提示。
4. 兵数を戦闘後の値へ更新（消えた分の隊列スロットを畳む）。
5. 閉じる（退場アニメは無し）。盤へ戻る。
