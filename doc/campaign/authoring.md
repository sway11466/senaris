# 冒険譚の制作方針（全キャンペーン共通）

すべての冒険譚（キャンペーン）に共通する作り方の取り決め。各冒険譚ファイル（`campaign/<冒険譚>.md`）はこの方針に従い、固有の内容（物語・casting・盤面）だけを書く。

---

## 会話パート（台本）

各ステージは**戦闘前と戦闘後に短い会話パート**を挟む。

- **構成**: 前（状況説明＋掛け合い）→ 戦闘 → 後（締め＋次回への引き）。戦闘後の終盤で**次ステージの教える要素を予告**し、ステージ間を1本につなぐ。
- **チュートリアル装置**: 前半は質問役が問い、指示役が答える形にして、操作のヒントを会話へ自然に織り込む（説明くさくなりすぎない範囲で）。
- **登場人物**: 各冒険譚で定義する。**役職ベースの没個性キャラ**（指示役・質問役・報告役など）を推奨。盤上ユニットと対応づけてよい。固有名は後付け可。
- **書式**: 引用ブロックで `> **話者**: セリフ`。ト書きは `> _（…）_`。台本は各ステージ詳細の「会話（戦闘前）／（戦闘後）」に直書きする。
- **データ化（実装済み）**: 会話の構造（誰が・どの顔で・どの順で）はステージJSONの `dialogue`（`{ intro:[...], outro:[...] }`、各行 `{ speaker, skin, text }`）に持つが、**presentation 専用**＝盤面データ(BattleState)には入れない（案P・`StageLoader.parse_dialogue`）。**テキストと話者名は翻訳キー**で持ち、実文字列は Godot 標準の翻訳CSV（正本 `data/i18n/dialogue.csv`＝`keys, ja, en…`）→ `tr()` で解決する（多言語対応）。**顔は `UnitSkin` の `portrait` スロット**（未用意は名前プレースホルダ）。表示は `ConversationPanel`（右エリアのチャット風・話者左右交互・「次へ」/「会話をスキップ」・戦闘後会話は勝利時のみ）。
