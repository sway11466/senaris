# ADR-0001: ゲームエンジンに Godot 4 を採用する

- **ステータス**: 承認済み (Accepted)
- **決定日**: 2026-06-07
- **対象**: ゲームエンジン / 開発基盤の選定

---

## コンテキスト

Nectaris（海外名 *Military Madness*）系の **ヘックス制ターン戦術 SLG** を、ファンタジー舞台で開発する。
このジャンルはターン制でグラフィック負荷が低く、盤面ロジック（移動・地形・包囲計算）が本体になる。

制約・目的は以下のとおり:

- **趣味目的**。ゲーム作りそのものの学習も兼ねる。
- **Steam（PC）配布**を実現したい。可能ならモバイルストア展開も視野。
- 描画負荷より **作りやすさ・続けやすさ・所有権の安定** を重視したい。
- 2D 中心。高品質 3D は不要。

候補: **Godot 4** / Unity / Unreal Engine / Web 完結（自前配布・Electron 等）。

---

## 決定

**Godot 4（GDScript・2D）を採用する。** Steam 連携が必要になった段階で **GodotSteam** アドオンを利用する。

---

## 根拠と影響 (Consequences)

### 採用する利点

- **完全オープンソース（MIT）/ 無料 / ロイヤリティ・ランタイム料なし。** 権利がコミュニティにあり「ある日突然規約が変わる」リスクが構造的に低い。
- **1 プロジェクトから Windows / macOS / Linux（＝Steam 対応）＋ iOS / Android ＋ Web に書き出せる。** Steam もモバイルも同一エンジンでカバーできる。
- **軽量で起動・反復が速く、2D とタイルマップに強い。** ヘックスタクティクスとの相性が良い。
- **GDScript は習得が速い**（Python 風）。学習目的に向く。基礎概念（シーン/ノード設計・状態管理・タイルマップ・入力処理）は他エンジンにも転用が効く。
- **成長中で実績もある。** 新規インディー採用率は約 11%、ゲームジャムでは Unreal を抜いて 2 位、Steam 上の Godot 製ゲームは mid-2026 時点で 35,000 本超。商用大ヒット例として *Slay the Spire 2*（2026 年 EA、同接 10 万超）が Godot 採用。

### トレードオフ・留意点

- **アセット/プラグインのエコシステムは Unity より小さい。**
- **家庭用機（コンソール）サポートは弱い。** → 現時点で対象外なので問題なし。
- **求人ボリュームは Unity / C# 優勢。** → 趣味目的のため許容。就職を重視する場合は再検討の余地。
- **高品質 3D の上限は Unity / Unreal より低い。** → 2D プロジェクトのため無関係。
- **iOS ビルドには Mac と Apple Developer 年会費（$99/年）が必要。** → モバイルは後回しのため当面は影響なし。

## 検討した代替案

| 候補 | 強み | 不採用の理由 |
| --- | --- | --- |
| **Unity** | 最大級のエコシステム、最多の対応プラットフォーム、最大の求人、成熟した C# 環境 | 今回の規模に対して重く、有料ティアと 2023 年のランタイム料騒動による不確実性。2D 趣味開発にはオーバースペック。 |
| **Unreal Engine** | 高品質 3D で最強、AAA 帯で収益シェア大 | 2D 戦術 SLG には過剰。学習コストと C++ の敷居が高い。 |
| **Web 完結（自前 / Electron）** | 最速のプロトタイピング、ブラウザ配布が容易 | Steam へのパッケージングが煩雑で本命ルートに不向き。ただし *設計検証用のサンドボックス* としては有用なので、その用途では残す。 |

---

## 補足

- 開発方針: まず **コアの面白さ（支援＝包囲システム）を最小プレイアブルで検証** し、手応えを得てから肉付けする。
- 参照: Godot 最新安定版は 4.6.3（2026-05）。Steam Direct は 1 タイトル $100（売上 $1,000 で返金）。
