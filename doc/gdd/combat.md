# 戦闘システム

ターン制ヘックス戦術シミュレーション（ファンタジー）の戦闘解決の設計メモ。
ゲームシステムの核はネクタリス（Military Madness）を踏襲しつつ、表現はオリジナルに置き換える方針。

> 関連: ユニット性能は [units.md](units.md)／マップ・占領は [map.md](map.md)／セーブは [../tech/gamesystem.md](../tech/gamesystem.md)／世界観は [world.md](world.md)。コンセプト上の位置づけは [../concepts.md](../concepts.md)。

---

## 設計の前提

- 核となるシステムはネクタリスを踏襲する（ターン制・ヘックスマップ・戦術級）。
- 知財方針：**メカニクス（ルール／仕組み）は流用、表現（名前・世界観・グラフィック・音・具体的なマップ配置）は総入れ替え**。
  - ネクタリスの権利は現在コナミ側にあるため、デッドコピーに見えない作りを徹底する。
- 世界観はファンタジー。ユニットの名前と見た目はオリジナルだが、**性能的な特性は引き継ぐ**。

---

## コアメカニクス

### 1. 生産なし（踏襲）

- ユニットの生産・補充は行わない。マップごとに与えられた戦力でやりくりする。
- 結果として **1体1体が貴重** になり、損耗そのものが戦略的な重みを持つ。
- 後述の「陣形」でユニットを消費する判断も、この縛りがあることで意味を増す。

### 2. 戦闘補正（包囲・支援・地形）

戦闘解決は **ネクタリス方式を踏襲しつつ、本作は決定的（乱数なし）** とする。基本ステータスに各補正を適用し、攻撃力と防御力の比から損害を**確定的に**算出する。両軍は同時に攻撃する。

> **決定的にする理由【確定】**: 「盤面で噛み合わせて勝つ」コンセプトは詰将棋的な読みが核。乱数は読みを濁らせ、生産なしで貴重な1体が運で溶けるのはフラストレーション（原典ネクタリスは "wide RNG" だが、これは現代基準ではむしろ弱点と判断）。決定的なら包囲・支援・地形・陣形がそのまま実力になり、**中断セーブはシード不要・リプレイ完全一致・AI先読みも正確**（アーキの未決「戦闘の決定性」を決定的で閉じる）。代償として AI が弱いと作業ゲー化するため、**強敵ユニット・マップ構成・将来の高度AI**で担保する。原典の「同時攻撃・相討ちあり」は決定的でも成立するので残す。

> **原典確認（出典: StrategyWiki / Wikipedia / Nintendo Life）**: ネクタリスの戦闘は **先制なしの同時攻撃**。防御側は同時に反撃し、損失は戦闘前の強さを基準に双方へ同時適用されるため、**1回の戦闘で両者が全滅（相討ち）し得る**のは仕様。**例外＝間接攻撃（砲撃系）には反撃が無い**（→ 本作のレンジ攻撃を入れる際は「間接攻撃は反撃なし」とする）。勝敗条件は [map.md](map.md#勝敗条件) 参照。

> **数値の扱い**：ネクタリスの補正値を **初期ベースライン** として採用してよい（バランス数値はルール＝システム側で、著作権の保護対象ではない）。ただし本作はロスターを取捨選択するため、最終的には自作ユニットに合わせて**再チューニング前提の叩き台**とする。

#### 用語【確定】

- **ユニット** … 盤上の駒＝小隊1個。
- **兵数** … 1〜8。ユニットの残存兵数（8=満員、1=壊滅寸前、0=消滅）。原典の「強さレベル」に相当。
- **ユニット攻撃力 / ユニット防御力** … 兵1体あたりの基礎攻撃力・防御力。原典の BuA/BuD に相当。

#### 実効ステータス【確定】

各補正を **乗算（経験・包囲・地形）＋ 加算（支援）** で適用する:

```
実効攻撃力 A ＝ 兵数 × ユニット攻撃力 × 経験 × 包囲 × 地形(攻) ＋ 支援(攻)
実効防御力 D ＝ 兵数 × ユニット防御力 × 経験 × 包囲 × 地形(防) ＋ 支援(防)
```

#### ダメージ算出（決定的）【確定】

同時攻撃。**戦闘前の兵数**で2方向を別々に計算し、両方を同時に適用する:

```
（X が Y を攻撃する場合）
Y失う兵数 ＝ clamp( round( k × Y兵数 × Aₓ^p / (Aₓ^p + D_y^p) ), 0, Y兵数 )
X失う兵数 ＝ clamp( round( k × X兵数 × A_y^p / (A_y^p + D_x^p) ), 0, X兵数 )   ← 反撃: Yの攻撃力 vs Xの防御力
```

- `A^p/(A^p+D^p)` ＝ 相手の兵数のうち削る割合(0〜1)。**互角(A=D)は常に0.5**、戦力差がついた時だけ鋭くなる。防御0でも発散しない。
- **k（殺傷力）**: 全体の削り量。大きいと短期決戦、小さいと削り合い（位置取りの手数が増える）。
- **p（決定力）**: 戦力差の効き。**初期 p=2**（攻撃2倍→0.8 / 半分→0.2）。互角は p によらず常に0.5。
- 攻撃力と防御力は**独立**。攻撃を強化しても被反撃は変わらない（同じ防御力＝同じ被ダメージ）。
- **最低ダメージ（膠着回避）【暫定】**: 兵数が少なく互角だと round が0になり膠着し得るため、暫定で「**A≥D なら最低1減**」。要チューニング。
- **間接攻撃（砲撃・弓系）は反撃なし**（防御側は反撃ダメージを受けない）。

> k・p は実装後に実際の手触りで調整する。式は `domain/combat/combat.gd` に集約し、純関数として（エンジン起動なしで）テストする。

#### ① 経験値補正（乗算）

`経験 ＝ 1 ＋ 下表の%`（例: Lv3 攻撃なら ×1.20）。原典%を初期ベースラインとする。

| Lv | 攻撃(+%) | 防御(+%) |
|----|----------|----------|
| 1  | +4%      | +5%      |
| 2  | +10%     | +10%     |
| 3  | +20%     | +20%     |
| 4  | +30%     | +30%     |
| 5  | +40%     | +40%     |
| 6  | +60%     | +60%     |
| 7  | +100%    | +100%    |
| 8  | +100%    | +100%    |

- 経験値の入手：撃破で攻撃側 +2／損傷（撃破せず）で両者 +1／無傷で防御側 +2。
- （別途、歩兵の占領で +4 という記述もあり。）

#### ② 包囲効果（乗算・被包囲ペナルティ）【確定: 段階式（本作アレンジ）】

- ファンタジー世界観に合わせ **「包囲」** と呼称（ZOCは一般的なウォーゲーム用語のため、リネームは法的な必要ではなくデザイン上の選択）。
- **囲まれた側の攻撃力・防御力が低下するデバフ**。式では「そのユニット自身」に掛かる自己係数 `包囲`。
  - ※「自分の攻撃が上がる」のではなく「**囲まれた敵が弱る**」点に注意。攻防どちらにも同じ係数が乗る。
- **原典はオール・オア・ナッシング（全周囲が覆われて −50%）だが、本作は段階式**にする（決定的ゲームなので、配置の工夫が連続的に報われる方が読み合いが深い）。

**判定（対象の周囲6ヘックスを分類）**:
- **占有** … 敵ユニットがそのヘックスにいる
- **ZOC下** … 空きだが敵に隣接している（敵の支配下）
- **盤外ヘックスは数えない**（盤端・隅は包囲されにくい＝防御的退避）

**ゲートと係数**:
```
隣接する敵ユニット < 2  → 包囲 ＝ 1.0（包囲不成立。1v1の通常攻撃では発動しない）
隣接する敵ユニット ≧ 2  → 包囲 ＝ clamp( 1 − (0.15 × 占有数 ＋ 0.05 × ZOC数), 0.10, 1.0 )
```
- **占有はZOCの3倍重い**（0.15 vs 0.05）＝ ユニットで密に塞ぐほど強い。
- **2体以上の隣接で初めて成立**。同じ2体でも離して挟むほど ZOC で覆う範囲が広がり強くなる。
- 攻撃能力のないユニットでも、隣接すれば包囲の頭数になる（→陣形で行動完了したユニットも盤面に居る限り寄与）。
- **直接攻撃のみ**。間接攻撃には乗らない。

代表値（係数）:

| 状況 | 占有/ZOC | 包囲 |
|---|---|---|
| 敵1体だけ隣接 | 1 / – | ×1.00（不成立） |
| 隣り合う2体 | 2 / 2 | ×0.60 |
| 1つ飛び2体 | 2 / 3 | ×0.55 |
| 対角2体（最小包囲） | 2 / 4 | ×0.50 |
| 3体で囲む | 3 / 3 | ×0.40 |
| 6体で完全に塞ぐ | 6 / 0 | ×0.10（下限） |

- 数値（占有0.15・ZOC0.05・ゲート2・下限0.10）はチューニング対象。実装は `domain/surround/surround.gd`。

#### ③ 支援効果（加算）【確定: 原典方式】

- **直接攻撃時**の底上げ。実効ステータスに**加算**される（乗算ではない＝味方が多いほど効く）。
  - **支援(攻)** ＝ Σ「**防御対象に隣接する自軍**」の `兵数 × ユニット攻撃力 × 0.5`
  - **支援(防)** ＝ Σ「**攻撃者に隣接する自軍**」の `兵数 × ユニット防御力 × 0.5`
  - 支援量は支援側の**兵数に比例**（弱った味方の支援は小さい）。経験値補正は含めない素の値。
- **防御側の上限**: 支援後の実効防御力は **支援前の実効防御力の2倍** まで。
- **直接攻撃のみ**。間接攻撃には乗らない（隣接していれば味方に支援を与える側にはなれる）。

#### ④ 地形効果（乗算・攻防別）【確定: 本作オリジナル拡張】

- **原典は防御のみ。本作は攻撃にも地形係数を持たせる**（攻防別の2係数）。罠地形＝攻撃強化、穴ぐら＝防御強化…のように地形の性格づけで戦略を広げる狙い。
- 各地形が `地形(攻)` と `地形(防)` を持つ。例（叩き台）:

| 地形 | 地形(攻) | 地形(防) | 性格 |
|---|---|---|---|
| 平地 | ×1.0 | ×1.05 | 無難 |
| 罠地形 | ×1.5 | ×1.0 | 攻めに強い |
| 穴ぐら | ×1.0 | ×1.5 | 守りに強い |
| 開けた荒野 | ×1.2 | ×0.8 | ハイリスク |

- 地形タイプ別の具体係数は map.md と連携して詰める。【未決】

#### 数値の扱い

- 補正値はネクタリスを初期ベースラインとし、自作ロスター確定後に再調整する（k・p・地形係数・経験表を含む）。


### 3. 陣形（本作オリジナル）

マップ上で特定のユニット配置が揃うと、スペシャルアクションが発動できる本作独自の概念。

#### 発動条件：ユニット種別に依存する「編成レシピ」

- **同種型**：同じ種別のユニットが規定数・規定の隣接で揃うと発動。
  - 例）ソーサラー×3 が隣接 → 大魔法
- **異種型**：異なる種別の組み合わせで発動。
  - 例）ソーサラー ＋ アーチャー → 魔法弓攻撃
- レシピが増えるほど、盤面構築と読み合いが豊かになる。
- **仮レシピは保留**：現在の例（ソーサラー×3→大魔法／ソーサラー＋アーチャー→魔法弓）は、ユニット確定後に練り直す前提。ソーサラー＝重量級術者、アーチャー＝弓系が確定したので再設計の材料は揃った。【保留】

#### 発動ルール

- 発動は **プレイヤーの明示操作**（条件が揃っても自動では出ない）。
- 発動に参加したユニットは **行動完了** になる。
  - → 1体のユニットは **1ターンに1つの陣形にしか参加できない**。
  - → 共有ユニット問題（複数の陣形条件に同時該当する状況）が、追加ルールなしで自然に解決する。
  - → 「この3体を大魔法に回すか、魔法弓やほかの行動に残すか」という取捨選択が毎ターン発生する。

### 包囲と陣形の関係

- **包囲＝下地（受動・連続）／陣形＝決め技（能動・離散）** という役割分担で噛み合う。
- 両者は微妙に引っ張り合う：
  - 包囲は「敵の周囲に味方を固める」ほど強い。
  - 陣形は「味方同士で特定の形を作る」必要がある。
  - 同時に満たしにくい局面が、そのまま戦術的な読み合いの深さになる。
- さらに、陣形のために固めた味方は敵に包囲されやすくもなる（攻守の表裏）。
