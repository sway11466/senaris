# 敵AI（思考パターン）

敵の思考を「**思考の軸（パラメーター）の組み合わせ**」で定義し、その組に**ラベル**を付ける。割り当ては**部隊（squad）単位**。

> 関連: 戦闘解決（決定的・包囲/支援/地形/経験）は [combat.md](combat.md)／移動・再移動は [movement.md](movement.md)／ステージデータ・占領兵(`can_capture`)は [map.md](map.md)。

---

## 基本方針

- **AI＝思考パラメーターの組み合わせ**。「突撃AI」などのラベルは、各軸の値を選んだ**プリセット**にすぎない。新パターンは軸の値を選んで名前を付けるだけで増える。
- **割り当ては部隊単位**。ステージデータで「敵」をフラットに並べず、**部隊（AIを共有するユニットの束）**を定義し、部隊ごとにAIラベルを割り当てる。
- **決定的（乱数なし）を活かす**: 戦闘結果が確定的（[combat.md](combat.md)）なので、「確殺」「最大与ダメ」「良トレード」は推定でなく**厳密に計算できる**。AIの強さは実質「どこまで先を読むか」に帰着する。

---

## 思考の軸（パラメーター）

### A. 起動（engage）— このターン動くか
- **突撃**: 常時、敵へ前進・交戦。
- **待機**: 索敵半径 `sight` 内に敵が入る／部隊が警戒発火するまで動かない。発火後は他軸に従って行動。

### B. 標的選択（target）— 誰を攻撃するか
攻撃可能な敵から、次の優先度で選ぶ（複数指定は順に tiebreak）:
- **近い**（最短接近）／**弱い**（残兵少）／**確殺**（この攻撃で倒せる）／**最大与ダメ**／**最小被ダメ**（反撃が軽い・間接優先）／**脅威優先**（射手・術者・高火力）／**攻撃力高い**／**防御力高い**。

### 占領兵優先（prefer_capturer）— 独立軸（ON/OFF）
- **ON**: 今ターン攻撃可能な範囲に**占領兵**（相手の `can_capture` ユニット）がいれば**最優先で狙う**（Bに優先）。いなければBに従う。
- 占領兵（クレリック等）は拠点を奪い戦況を変える高価値なので、潰しにいくか無視するかを独立して選べる。Bとは別レイヤーの前処理。

### C. 位置取り（position）— どのhexから当てるか
- **直進**（最短で隣接/射程へ）／**包囲**（味方と隣接して包囲補正を狙う）／**地形**（高所・防御地形を取る）／**射程維持・離脱**（間接＋再移動でヒット&アウェイ）／**退避**（瀕死は下がる）。

### 前進距離（advance）— 動く時どこまで詰めるか
- **最大前進**: 移動力の許す限り標的へ詰める（攻撃できないターンも前へ）。
- **安全前進**: **次の敵ターンに攻撃されない位置**で止まる（相手の脅威圏＝敵の移動力＋射程の外）。じわじわ詰めて被弾を避ける。射程ユニットの「射程維持・離脱」とも噛み合う。
- 攻撃できるターンは標的を殴る位置（C）が優先。advance は主に「**今ターン攻撃できない時の前進量**」を決める。

### D. リスク（risk）— 不利でも踏み込むか
- **無謀**（損得無視で攻撃）／**慎重**（悪いトレードは見送り、瀬戸際の兵数で退く）。

### 部隊プロパティ（squad単位）
- **sight（索敵半径）**: 待機の起動距離。
- **alert（一斉警戒）**: `none`／`squad`（部隊内の誰かが発見・交戦されると全員起動）。
- **focus（集中）**: 部隊で同じ標的へ攻撃を集中するか。

## 決定モデル（実装の核）

1ユニットの手番は「**候補（移動先 × 標的）を列挙 → スコア付け → 最良を選ぶ**」。上の軸がスコア関数を決める。

- A起動が「待機・未発火」なら何もしない（候補列挙に進まない）。
- 占領兵優先ON かつ攻撃可能な占領兵あり → その占領兵を標的に固定し、C/D で位置を選ぶ。
- それ以外は B で標的、C/D で位置を決める。
- 攻撃可能な標的が無い手番は、**前進距離（advance）**に従って前進のみ（最大前進／脅威圏外で停止）。
- 決定的なので各候補の与/被ダメは厳密計算できる。

|---|---|---|---|---|---|---|---|---|

## データ構成

### AIプリセット＝データ駆動（CSV正本）

- **軸ロジック（各軸の中身の計算）はコード**、**組み合わせ（ラベル×軸の値）はデータ**＝ハイブリッド。「軸の動き方」はコードに実装し、「どの軸をどう組むか」は表で持つ。
- `data/ai/ai.csv`（正本・2行ヘッダ）→ `ai.json`（生成）。terrain/movement と同じ CSV正本→JSON生成パイプラインに乗せる（[units.md](units.md) のデータ構成参照）。
- コードは「パラメーター化された1つの Brain」がプリセットを読んで振る舞う。現状の `nearest_attacker_brain` は **charge** プリセットへ統合・リネームが移行の筋。
- 実装は `domain/ai/`（純ロジック・Godotノード非依存）。

### 部隊（squad）＝ステージデータ

ステージJSONで部隊を定義し、AIラベルを参照する:

```jsonc
"squads": [
  { "name": "手下部屋", "ai": "guard", "sight": 3, "alert": "squad",
    "units": [ {"type":"goblin", "col": .., "row": ..}, ... ] },
  { "name": "ボス部屋", "ai": "guard", "alert": "squad",
    "units": [ {"type":"goblin_lord", ...}, ... ] }
]
```

- 部隊が `units` を内包。`ai` はプリセットのラベル参照。`sight`/`alert` 等で部隊ごとに上書き可。
- AI不要のユニット（味方・単独の敵）は従来どおり `units` 直書きでよい。部隊は「敵の行動定義のための任意の括り」。
