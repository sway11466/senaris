# 敵AI（思考パターン）

敵の思考を「**思考の軸（パラメーター）の組み合わせ**」で定義し、その組に**ラベル**を付ける。割り当ては**部隊（squad）単位**。

---

## 基本方針

- **AI＝思考パラメーターの組み合わせ**。「突撃AI」などのラベルは、各軸の値を選んだ**プリセット**にすぎない。新パターンは軸の値を選んで名前を付けるだけで増える。
- **割り当ては部隊単位**。ステージデータで「敵」をフラットに並べず、**部隊（AIを共有するユニットの束）**を定義し、部隊ごとにAIラベルを割り当てる。
- **決定的（乱数なし）を活かす**: 戦闘結果が確定的（[combat.md](combat.md)）なので、「確殺」「最大与ダメ」「良トレード」は推定でなく**厳密に計算できる**。
- **先読みは1手**: 今ターンの戦闘を厳密評価＋次の敵ターンの被攻撃（threat）まで見る。決定的なので1手でも精度が高い。より深い読みは将来の高度AIで。

## 思考の流れとパラメーター

### 1. 起動（engage）— このターン動くか
- 必須
  - 兵数： 兵数が減少している場合は確定で動く
- 選択（複数選択可能 OR/ANDを選べる）
  - 突撃: 常時、敵へ前進
  - 間合: 索敵半径 `sight` 内に敵が入ったら動く
  - ターン数: 指定のターン数が経過したら動  く
  - 部隊: 部隊の誰かが動いたら動く

### 2. 占領（capture）— 取れる拠点があれば取る
- 必須（パラメーター無し・全AI共通の既定＝CSVに列を持たない）
  - **占領可ユニット（`can_capture`）のみ**。今ターンの移動範囲に**自陣営以外の拠点**（敵・中立）があれば、そこへ移動して占領する。**攻撃より優先**。複数あれば最寄り。

### 3. 撤退（retreat）— 撤退するかどうか？
- 必須
  - 兵数: 兵数が指定した基準値を下回ったら撤退する、ただし、自軍の拠点がない場合は撤退しない

### 4. 攻撃（attack）— 攻撃するかどうか
- 必須
  - なし
- 選択（複数選択可能 OR/ANDを選べる）
  - 突撃: 攻撃可能なら必ず攻撃する
  - 獲物のみ: 獲物（下記「弱者狙い（weak）」の対象）か、確殺できる相手が射程内にいるときだけ攻撃する。硬い前衛しかいなければ殴らずに前進を続ける
  - 単独有利: 総合力で有利な相手ユニットがいる場合（移動開始時点で実効攻撃力＋実効防御力が勝っているユニットがいる）
  - 包囲可能: 味方ユニットがいて包囲可能な場合（あとから包囲可能なら単独でも攻撃をする）
  - 包囲状態: すでに味方ユニットが隣接していて包囲効果が発動する場合
  - 反撃なし: 反撃を受けないことが確定している相手がいる場合
  - 確殺: 一撃で倒しきれる相手がいる

### 5. 攻撃対象（target）— 攻撃する場合の対象と位置の選び方
- スキップ
  - 4.で攻撃することが選択されていない場合はこの処理は行わない
- 必須
  - 間接攻撃: 間接攻撃優先（間接攻撃できない場合は不成立）
- 選択（複数選択可能、優先順位順に並べて合致した行動をとる）
  - 最短距離: 一番近い
  - 弱者狙い: 倒しやすい敵を優先する（設計は下記「弱者狙い（weak）」）
  - 最大与ダメ: 攻撃時点で一番兵相手の数を減らせる
  - 最小被ダメ: 攻撃時点で一番兵自分の数が減らない
  - 占領兵: 占領できるユニットを優先して狙う
  - 間接兵: 間接攻撃できるユニットを優先して狙う
  - 飛行兵: 飛行しているユニットを優先して狙う
- 補足
  - 同じ条件が複数ユニット合致する場合は、合致したユニットで次の優先順位項目を使って判断する、すべての条件を使っても絞りきれない場合は、マップ番号が小さいものを対象とする

#### 弱者狙い（weak）の設計

「獲物」に固執して後衛の脆いユニットを狩りに行く動き。target 軸 `weak` を核に、attack 軸 `prey`・advance 軸 `flank` を組み合わせて表現する。**実装済み**。

- **獲物の定義**: 盤上の敵のうち**ユニット防御力（素のステータス）が最も低い**もの。同率は近い方 → id小。**兵数は見ない**（獲物がステータスで決まるので行軍が安定し、「あいつら馬車を狙ってる」とプレイヤーが読める。瀕死の敵は attack 側の確殺ルールが拾う）。獲物が倒れたら次に防御の低いユニットが自動的に次の獲物になる（毎回盤面から再計算）。
- **攻撃有無（attack=prey・獲物のみ）**: 射程内に獲物か確殺できる相手がいるときだけ攻撃する。硬い前衛と無駄に殴り合わず、殴られても獲物へ歩き続ける。
- **攻撃対象（target=weak）**: 攻撃候補が複数なら、戦闘式（[combat.md](combat.md)）で与ダメを厳密計算し**攻撃後の残兵が最小**の敵を選ぶ（確殺が自然に最優先）。同値は近い方 → id小。
- **前進（advance=flank・回り込み）**: 目標は獲物。移動範囲のうち**敵のZOC（敵に隣接するマス）に入らないマス**を優先して獲物への距離を縮める＝前衛の正面を避けて横へ滑る。安全なマスで距離が縮まらないときだけ通常の最大前進で詰める（獲物への最終接近＝獲物自身のZOCへの進入もこのフォールバックで成立）。間接攻撃の射程圏は避けない（避けると獲物に近寄れなくなるため）。
- 対処法はZOC: 前衛の壁（ZOCの帯）で足止めできる＝「後衛を守れ」をプレイヤーに教える圧になる。

### 6. 前進距離（advance）— 攻撃できない場合に、どこまで動くか？
- 必須
  - なし
- 選択（排他的選択）
  - 最大前進: 移動力の許す限り標的へ詰める（攻撃できないターンも前へ）。
  - 間合維持: 次の敵ターンに攻撃されない位置で止まる（相手の脅威圏＝敵の移動力＋射程の外）。じわじわ詰めて被弾を避ける。射程ユニットの「射程維持・離脱」とも噛み合う。
  - 部隊維持: 部隊が一丸となって動くため最小移動力のユニットに合わせて進む
  - 慎重前進: 部隊一丸かつ間合い維持で進む
  - 拠点前進: **全ユニットが最寄りの「占領できる拠点」へ向かう**＝部隊ごと拠点攻略に向かう動き。占領できるのは占領可ユニットだけで、護衛はそこで戦う（部隊に最低1体は占領兵を入れる想定）。拠点が無ければ通常の標的前進
  - 回り込み: 敵のZOCに入らないマスを優先して標的（弱者狙いなら獲物）へ詰める。安全なマスで距離が縮まらないときは最大前進で詰める（詳細は上記「弱者狙い（weak）の設計」）

### 7. 拠点出撃（deploy）— 拠点の控えを盤上に出すか

- **AIが制圧中（`base.team` が敵）の拠点だけ**が対象。控えのうち出せるのは **native ルールを満たす駒のみ**（native＝その拠点の陣営、または中立）＝奪われた敵駒は閉じ込めで出せない（[map.md](map.md) 出撃）。
- **いつ出すか＝起動(engage)を再利用**。拠点を「起きたら殴る代わりに増援を吐く眠りユニット」とみなし、engage トリガーを**拠点hex基準**で評価する:
  - `charge`＝出せるなら即（突撃）／`sight:N`＝拠点から距離N内に敵が来たら開始（待機・潜伏）。
- 起動したら、**出撃条件を満たす限り“出せるだけ”出す**（そのターンに空き隣接が埋まる／garrison が尽きるまで＝1ターンの実質上限は空き隣接の数）。出す先は最寄り敵に近い空きマスから。出した駒はそのターン行動完了で、**次から拠点の preset で戦う**（＝拠点は1つの squad）。
- **全体の上限**は garrison の頭数（敵AIは「入る」を使わない＝補充されない）。
- 実装状況: **実装済み**（`NearestAttackerBrain._try_deploy`＋`BattleState.deploy`/`can_deploy_garrison`／`AiAction.deploy`。engage 判定を拠点基準へ再利用。デバッグ: `data/stages/debug/spawn.json`・テスト `tests/unit/test_ai.gd`）。

## ラベル＝プリセット

プリセットの**正本は `data/ai/ai.csv`**（CSV正本→`ai.json` 生成）。doc にフル一覧は持たない（CSVが真実）。

- **列**: `label, name, engage, sight, retreat, attack, target, advance`（name＝表示名、以降＝思考の流れのパラメーター。**2.占領は全AI共通の既定なので列を持たない**）
- **セル表記**: `|`＝OR ／ `;`＝順序 ／ `-`＝該当なし（この軸で特別なことをしない、を表す値。空欄ではない）。「間接攻撃を優先（反撃回避）」は全AI共通の既定なので列に持たない。
- 一斉警戒は `engage` の `squad` トリガーで表現（専用列は持たない）。

収録:
- **charge**（突撃AI）: 常時前進・殴れれば殴る・近い敵・最大前進。st1〜5・7。
- **guard**（待機AI）: 索敵/部隊で起動（一斉警戒）・退かない・近い敵・最大前進。st6（ダンジョン）。**実装済み**（起動: sight／squad／被ダメ／自衛。`turn:N` は未実装）。デバッグステージ: `data/stages/debug/guard.json`。
- **raid**（拠点攻略AI）: 常時前進・殴れれば殴る・近い敵・**拠点前進**（部隊ごと占領できる拠点へ向かう）。本拠地占領で攻める敵に。デバッグステージ: `data/stages/debug/hq.json`。
- **weak**（弱者狙いAI）: 常時前進・**獲物（盤上最低防御）と確殺だけ殴る**・残兵最小・**回り込み**（ZOCを避けて獲物へ）。上記「弱者狙い（weak）の設計」参照。後衛の脆いユニットを守らせる圧になる。冒険譚2 st5・st6。**実装済み**。デバッグステージ: `data/stages/debug/weak.json`。

新ラベルは CSV に**行を足すだけ**で増やせる。

## データ構成

AIは2層: **プリセット（ふるまいの組み合わせ）＝CSV** と **割り当て（どの部隊がどのプリセットか）＝ステージデータ**。**各パラメーターの計算（軸の動き方）はコード**、**その組み合わせはデータ**というハイブリッド。

### プリセット ＝ `data/ai/ai.csv`（CSV正本）

- 正本 `data/ai/ai.csv`（2行ヘッダ）→ `ai.json` 生成。terrain/movement と同じ CSV正本→JSON パイプライン（[units.md](units.md) 参照）。**`.csv` は `importer="keep"` 必須**（Godot の翻訳化の罠を回避）。
- 1行＝1ラベル＝「思考の流れ」のパラメーター（2.占領は共通既定＝列なし）。列:

| 列 | 対応 | 値・表記 |
|---|---|---|
| `label` | ラベルid | `charge` / `guard` / `raid` … |
| `name` | 表示名（人間用） | `突撃` / `待機` / `拠点攻略` … |
| `engage` | 1.起動トリガー | `charge`／`sight`／`turn:N`／`squad`。複数は `|`＝OR（被ダメ起動は常時ON＝暗黙） |
| `sight` | 索敵半径（間合トリガー用） | 数値 ／ `-` |
| `retreat` | 3.撤退閾値（兵数。`0`＝退かない） | `0` / `2` … |
| `attack` | 4.攻撃条件 | `always`／`prey`（獲物のみ）／`solo_adv`／`surround_able`／`surrounded`／`no_retal`／`kill`。複数は `|`＝OR |
| `target` | 5.対象優先（順序） | `near`／`weak`／`maxdmg`／`mindmg`／`capturer`／`ranged`／`flyer`。順序は `;`（間接攻撃優先は既定＝暗黙） |
| `advance` | 6.前進 | `max`／`spacing`／`squad`／`careful`／`base`（拠点前進）／`flank`（回り込み） |

#### 既定と省略のポリシー

- ai.csv は全セルを明示する（省略なし）。engage・sight を含め全軸を必ず埋める。`-` は「該当なし」を表す値であって空欄ではない（例: sight `-`＝索敵トリガー不使用）。CSVを見れば全プリセットの全軸が値として読める＝一覧性を保つ。
- ステージJSONの squad はパラメーターを省略できる。省略＝そのプリセットの値を継承（解決順は `squad上書き ＞ プリセット`）。省略が自明に読めるのはこの一段だけなので、CSV側では省略を認めない。
- 既定は単一の定数へ集約。ai.csv 由来のプリセットは全軸そろい（生成時に検証済み）なので実データでは既定に落ちない。ただし部隊に属さない素のユニット・テストの部分プリセットには基準値が要るため、Brain は「素の charge AI」を表す単一の `DEFAULT_PRESET` 定数だけを持つ（以前 `_param` の各所に散っていた `"max"`/`"charge"`/`"always"` 等の既定リテラルはここへ集約＝ドリフト源を撤去）。「グローバル既定をゼロにする」ことは素のユニットが基準値を要するため不可能で、実現できるのは“散在の撤去＝単一化”まで。実装済み。
- 欠損はデータのバグとして扱い、プレイヤーに届く前に検出する。想定するのは「CSVのセル埋め忘れ」と「squad が存在しないラベルを参照」の2つ。検出箇所: `convert.gd` の生成時チェック（どのラベルのどの列かを出す）と、`AiCatalog` 読み込み／GUTテストでの全軸そろい検証。対戦中(runtime)の欠損チェックは不要（上記で保証済み）だが、保険として `_param` に「プリセットにキーが無ければ `assert`」を置くのはよい（export版では消えるのでコストなし）。これはCSV→データ生成全体の基本方針の一適用例（doc/tech/architecture.md「CSV→データ生成のバリデーション」）。

- コードは「**パラメーター化された1つの Brain**」がプリセットを読んで振る舞う（ラベルごとにクラスを作らない）。現状の `nearest_attacker_brain` は **charge** へ統合・リネームが移行の筋。
- 実装は `domain/ai/`（純ロジック・Godotノード非依存）。
- **実装済み（`nearest_attacker_brain`）**: 2.占領ステップ（占領可ユニットが移動範囲内の敵/中立拠点を攻撃より優先して取る）と、前進の `base`（拠点前進＝全ユニットが拠点へ向かう）オプション。
- **起動（engage）も実装済み＝待機AIが動く**: `charge`（常時）／`sight`（索敵半径内に敵）／`squad`（部隊の誰かが起動＝一斉警戒）／被ダメ（`BattleState.attack` が確定起動）／自衛（射程内に敵が来たら起動＝隣で寝続けない）。起動済みフラグは `BattleState`（`is_engaged`）に持ち、一度起動したら戻らない。`turn:N` は未実装。
- **CSVパイプライン実装済み**: `data/ai/ai.csv` →（`convert.gd`）→ `ai.json` → `AiCatalog.load_default()`（data層・純データ）→ `NearestAttackerBrain.from_preset(p)`（domain層でBrain組立）。
- **部隊(squad)割り当ても実装済み**: ステージJSONの `enemy`＝部隊の配列（下記）を StageLoader が読み、BattleState が unit→部隊の対応を保持。**敵は必ず部隊に属する**（バラ配置は無い）。Brain はユニットごとに「部隊の上書き ＞ 部隊プリセット ＞ Brain既定プリセット ＞ `DEFAULT_PRESET`」で解決する（`ai` 未指定/未知の部隊＝charge 相当＝`DEFAULT_PRESET`。散在していた既定リテラルは単一定数へ集約済み。未知ラベルの明示検出は将来）。デバッグステージ: `data/stages/debug/hq.json`（raid部隊）／`data/stages/debug/guard.json`（guard部隊×2）。プリセットのうち効くのは `engage`・`sight`・`advance`（`max`/`base`/`flank`）・`attack`（`prey` のみ。ほかは既定＝殴れれば殴る）・`target`（`weak` のみ。ほかは既定＝射程内の兵数最小）。retreat は未配線＝現状は常に退かない（CSVに書いても効かない。配線は将来対応）。

### 割り当て ＝ ステージデータ（部隊）

ステージJSONの `enemy` セクションで**部隊（squad）**を定義し、AIラベルを参照する。`sight`/`retreat` などはプリセット既定を**部隊ごとに上書き**できる（陣営はセクションで決まるので駒に `team` は書かない）:

```jsonc
"enemy": [
  { "name": "手下部屋", "ai": "guard", "sight": 3,
    "units": [ {"skin":"goblin", "col": .., "row": ..}, ... ] },
  { "name": "ボス部屋", "ai": "guard", "retreat": 0,
    "units": [ {"skin":"goblin_lord", ...}, ... ] }
]
```

- 部隊が `units` を内包。`ai`＝プリセットのラベル参照。数値パラメーターは部隊で上書き可（無指定はプリセット既定）。
- **一斉警戒**は `engage` の `squad` トリガーで成立（部隊の誰かが起動 → 全員起動）。専用フィールドは持たない。
- AI不要のユニット（味方・単独の敵）は従来どおり `units` 直書きでよい。部隊は「敵の行動定義のための任意の括り」。
- **拠点も `ai` を持てる**＝garrison を出す preset を拠点=squad として指定する（`bases[].ai`＋`sight` 等の上書き）。トリガ成立で控えを出し（[§7 拠点出撃](#7-拠点出撃deploy-拠点の控えを盤上に出すか)）、出した駒はその preset で戦う。拠点データ自体は [map.md](map.md)。
