# BGM（音楽制作・組込）

BGM の制作方針と組込の運用。**費用をかけない**前提で、限られた戦力を盤面で噛み合わせて勝つ本作らしく、少ない曲を作り込んで回す。翻訳アセットの運用（[../tech/i18n.md](../tech/i18n.md)）と同じく「素材パイプラインの取り決め」を扱う。

## 前提

- **予算ゼロ**。有料の音源・外注は使わない（無料の範囲で完結させる）。
- 制作者はゼロから作曲はしないが、**楽譜が読め、ベースのメロディがあれば編曲・修整ができる**。
- 配布はまず Steam（PC）＝**商用**。使う素材はすべて商用利用可であることを確認する。

## 方針（二段構え）

曲を全部自作しようとしない。用途で手段を分ける。

- **通常曲（大半）**: 商用可のフリー BGM 素材を**そのまま使う**。選ぶだけ。
- **看板曲（数曲）**: タイトル／メインテーマ／ボス戦など、作品の顔になる曲だけ **MuseScore で自作アレンジ**する。
- 元手を増やさないため、**メインテーマの編成違い**（タイトル版・ボス版など）で流用し、統一感を出す。盤面同様「与えられた戦力を噛み合わせる」。

## ツール（すべて無料）

- **Muse Hub** — Muse 社の配布・更新ランチャー。下記を入れる入口。
- **MuseScore Studio**（旧 MuseScore 4）— 楽譜作成ソフト本体。制作・編集はこれ。楽譜が読める制作者に最も手が合う。
- **Muse Sounds** — 無料の高品質音源（オーケストラ等）。Muse Hub 経由で入れる。これで書いた曲がぐっと良く鳴る。
- 必要になったら無料 VST（Spitfire LABS / BBC SO Discover 等）＋無料 DAW（Cakewalk / Reaper / LMMS）に発展。ただし**まずは MuseScore 単体で完結**させる。

## データ形式

| 形式 | 拡張子 | 役割 | 扱い |
|---|---|---|---|
| MuseScore 原本 | `.mscz` | 編曲の原本 | **必ず残す**（後で直せる） |
| MusicXML | `.musicxml` | 記譜の交換フォーマット | ベースメロディの受け渡し・編集の入口 |
| MIDI | `.mid` | 演奏データ | フリー MIDI や生成物の取り込み・プレビュー |
| Ogg Vorbis | `.ogg` | 最終成果物 | Godot に載せる。原本から作り直せる生成物 |

- **原本 `.mscz` が正**。`.ogg` は書き出し（生成物）だが、絵の PNG（`-src` からの生成物）と同じく**コミットする**（置くだけで鳴る運用のため）。
- **`.ogg` の中身は Ogg Vorbis 必須（罠）**: `.ogg` はコンテナ名で、中身が **Opus** のファイルも同じ拡張子を名乗る。Godot 4 が読めるのは **Vorbis のみ**（Opus はインポートで `valid=false`）。書き出し・変換ツールによっては Opus を吐くので、投入前に `head -c 64 x.ogg | strings` で `vorbis`（OK）/`OpusHead`（NG）を確認する。NG なら MuseScore から WAV を書き出し直し `ffmpeg -i in.wav -c:a libvorbis -q:a 6 out.ogg` で変換（Opus からの再エンコードは劣化が重なるため WAV 経由推奨）。
- **書き出しの基準値**: WAV（中間素材）＝ **44100 Hz／16bit PCM／ステレオ**（Godot の既定ミックスレートが 44100 Hz のため合わせる。48kHz は実行時リサンプルになるだけで利点なし）。OGG Vorbis（最終）＝ **44100 Hz／`-q:a 6`（≈192kbps）**。MuseScore のエクスポート設定ではサンプルレート 44100 Hz を選択、ビット深度は既定の 16bit のままでよい。
- 他所からメロディをもらう時は **MusicXML か MIDI** で受け取り、MuseScore で開いて直す。MIDI→楽譜変換はリズム解釈がずれることがあるので取り込み後に手直し前提。

## 制作ワークフロー

```
入手（PD楽譜 .musicxml / フリー .mid）
  → 編曲（MuseScore、原本 .mscz で保存）
  → 書き出し（.ogg）
  → Godot 組込（AudioStream 登録・ループ設定）
```

## ベースメロディの入手（作曲しない前提の要）

商用の安全さを最優先し、**権利がクリーンな順**に使う。

1. **パブリックドメインのクラシック**（IMSLP 等）— 楽譜（作曲）が著作権切れ。**自分で写譜・アレンジすれば成果物は自作物**になり最も安全。※他者の「演奏録音」「現代の編曲版」は別権利なので、古い原譜から起こす。
2. **伝統曲・民謡（PD）** — ファンタジー舞台と相性が良い。旋律だけ借りて再編曲。
3. **商用可のフリー MIDI／モチーフ集** — ライセンスを確認して土台に。
4. **AI 生成は補助まで**（Suno 等）。商用ライセンスがプラン依存で、AI 生成物の著作権自体が各国で未確定。**看板曲ほど自分の改変を厚くし**、素性を自作寄りに寄せる。

> 補足: 生成ツールに旋律のたたき台（MusicXML/MIDI）を出させ、MuseScore で大幅に編曲する運用も可。あくまで「ベース」で、良し悪しの判断と仕上げは制作者が行う。

## トラックライブラリ（ムード別・キャンペーン横断）

BGM は**ステージ単位**で流す。本作はマップそのものが戦場で1回の殴り合いが一瞬なので、**戦闘ごとに曲を切り替えず、攻撃・着弾は効果音（SFX）と戦闘演出で示す**（ネクタリス／Advance Wars 型。戦闘が別画面に遷移するファイアーエムブレム型とは分ける）。

**トラックはキャンペーンに属さない。** ムード（状況）別のライブラリとして育て、各ステージが JSON の `bgm` 欄で ID を選ぶ。同じようなマップ／状況なら別キャンペーンでも同じ曲を使い回す（FE 等の汎用マップ曲と同じ運用）。トラック ID もムード基準で命名し、キャンペーン接頭辞（`t1_` 等）は付けない。新しいムードが必要になったらライブラリに曲を足す。

| track_id | ムード（状況） | 雰囲気・主楽器 | ループ | 状態 |
|---|---|---|---|---|
| `title` | タイトル | 作品の顔。メインテーマ。オーケストラ | ○ | 未着手 |
| `menu` | メニュー／ステージセレクト | おおらか・爽やか・広がり。フルート＋弦アルペジオ | ○ | たたき台あり（32小節・ニ長調） |
| `journey` | 旅立ち・平原・冒険の始まり | 明るく前進。歩くベース＋速い和音変化。ニ長調 ♩=108 | ○ | たたき台あり（32小節） |
| `raid` | 襲撃・防衛戦・切迫 | 弦オスティナート＋ティンパニ。ロ短調 ♩=126 | ○ | たたき台あり（32小節） |
| `dungeon` | 洞窟・地下・探索 | ドローン＋まばらな旋律＋心拍。ホ短調 ♩=80 | ○ | たたき台あり（32小節） |
| `victory` | 勝利スティンガー | 短い高揚。数秒・ループなし | × | 未着手 |
| `defeat` | 敗北スティンガー | 短い沈み。弦／独奏 | × | 未着手 |
| （危機系） | 状態切替用の暗転版 | 元曲の短調化・低音・打楽器増（`journey_crisis` 等、元 ID＋接尾辞） | ○ | 未着手 |

割当例（チュートリアル1）: st1–2 = `journey` ／ st3–5 = `raid` ／ st6–7 = `dungeon`。

> 「戦闘曲」を独立トラックとしては持たない。緊張感は**ステージ曲の出し分け＋下記の状態切り替え**で作る。個々の戦闘は SFX（[../tech/combat_scene.md](../tech/combat_scene.md) の演出）で示す。
> 使い回しで耳が飽きてきたら、編成違いバリエーション（同じ `.mscz` から派生）や、キャンペーンの看板1曲だけ専用曲を足して対処する。最初から作り分けない。

## 管理・運用（ディレクトリ・トラックID・ステージ指定）

絵の管理（[../art/overview.md](../art/overview.md)、`assets/units/` ＋ `assets/units-src/`、autowire）と同じ思想で揃える。

### ディレクトリ — `xxx` / `xxx-src` の対

```
assets/
  bgm/                  ← ゲーム用 .ogg（Godot が読む）
    journey.ogg
    raid.ogg
  bgm-src/              ← 制作元
    journey/
      journey.mscz      ← 編曲の正
      journey.musicxml  ← 受け渡し用（任意）
    credits.md          ← 権利・ライセンス台帳（曲単位）
```

- 絵の「PNG＝`units/`、PSD・指示書＝`units-src/`」と同型。効果音を作る時は `assets/sfx/`＋`assets/sfx-src/` を並べる。
- 台帳は素材の隣（`units-src/{group}/style.md` と同じパターン）。

### トラックID — 規約 autowire

- 曲は**トラックID**（`journey` / `raid` / `title` / `victory` …＝上記ライブラリ表）で扱い、`assets/bgm/{track_id}.ogg` を規約で解決する（skin の画像 autowire と同型）。
- ファイルが**あれば**鳴る。**なければ無音＋開発ログ1行**（絵のプレースホルダの音版。ゲームは止めない）。曲が完成したら `.ogg` を置くだけで鳴り出す＝コード・データ変更なし。

### ステージ単位の指定 — ステージJSONの `bgm` 欄

```json
{ "name": "tutorial-st1",
  "bgm": { "main": "journey", "crisis": "journey_crisis" } }
```

- スロット制：`main`（必須）＋ `crisis`（任意＝状態切替用。未指定なら切替イベントが来ても曲は変わらない）。将来 `intro` 等もスロット追加で対応。
- **フォールバック連鎖**：ステージの `bgm` → `campaign.json` の既定 → 全体既定。全ステージに書かなくても冒険譚単位の指定で済む。

### レイヤー配置

| レイヤー | 責務 |
|---|---|
| `data` | トラックカタログ（ID→パス解決・autowire）。純ロジック |
| `application` | ステージ開始時に `bgm` 欄から「鳴るべきトラックID」を決定。`crisis` フラグ管理もここ |
| `presentation` | `BgmPlayer`（AudioStreamPlayer×2 のクロスフェード）。ID を受けて鳴らすだけ |

## Godot 組込

- 形式は **`.ogg`**（軽量・ループ設定しやすい）。`AudioStreamOggVorbis` で読み、`AudioStreamPlayer` で再生。
- ループは Ogg 側 or インポート設定の **`loop` / `loop_offset`** で管理（`.import` に乗る＝絵と同じ管理感覚）。ステージ中ずっと流れる曲はループ必須。
- レイヤー上、音の再生制御は presentation 寄り。曲の選択ルール（場面→曲）は application に薄く持たせ、`domain`/`data` は音に依存させない（[../tech/architecture.md](../tech/architecture.md)）。

## ループ設計の注意

- マップ曲などは延々流れるので、**終端が頭に自然につながる**よう末尾を整える。無音や余韻の尻切れに注意。
- MuseScore で書く段階から「ループ点」を意識し、最後の小節から 1 小節目への接続を確認する。

## 状態でBGMを切り替える（アダプティブ）

特定の状況で曲を切り替えられる。例: **敵が必殺技を使ったら危機BGMに変わり、以後クリア／敗北まで戻らない**。

- **方式**: まずは**2曲をクロスフェード**する単純切り替えで十分（1〜2秒）。音楽的にきっちり繋ぎたければ Godot 4 の **`AudioStreamInteractive`**（クリップ＋遷移ルール。次の小節頭で切替）を使う。レイヤー加算（`AudioStreamSynchronized`）は素材コストが高いので後回し。
- **「永続」＝状態フラグ1個**: ステージ状態に `crisis: bool` を持ち、一度立てたら戻さない（曲がパタパタ切り替わる事故を防ぐ）。ステージ開始でリセット。
- **責務分担**（[../tech/architecture.md](../tech/architecture.md)）:
  - `domain` … 「敵が必殺技を使った」という**事実をシグナル発火**するだけ（音を知らない）。
  - `application` … それを受けて BGM 状態を決める（`crisis` を1回だけ立てる）。
  - `presentation` … `AudioStreamPlayer` の差し替え／クロスフェードを実行。
- **危機BGMの用意**: 新曲は不要。**通常曲を短調化／テンポ上げ／低音・打楽器増**した“同テーマの編成違い”を同じ `.mscz` から派生（元手ゼロ）。ループ点整備は両曲に必要。
- **セーブ**: 途中セーブがあるなら `crisis` フラグも保存する（ロード時に通常曲へ戻る事故を防ぐ。[../tech/gamesystem.md](../tech/gamesystem.md)）。
- **引き金の粒度**: 切替は“盤面が変わる級”のイベント（必殺技・ボス出現など）に絞る。弱いイベントで変えると唐突。

## 権利・ライセンス台帳

商用（Steam）前提。使った素材は**必ず台帳（`assets/bgm-src/credits.md`）に記録**する。有料データ保護の方針（[../adr/ADR-0002-paid-data-protection.md](../adr/ADR-0002-paid-data-protection.md)）とは別に、**こちらは「他者の権利を侵さない」ための管理**。

記録項目（曲ごと）:

- 曲名（本作での用途）／出典（サイト・IMSLP番号・PD原譜 等）
- 入手形式（PD楽譜／フリーMIDI／自作 等）
- ライセンス（商用可否・改変可否・**クレジット表記の要否**）
- 改変度（そのまま／編曲／自作）

## クレジット表記

- 表記が必要な素材は、**Steam ページとゲーム内クレジット**にまとめて記載する。
- 表記文はライセンスの指定文をそのまま使う（自己流に変えない）。
