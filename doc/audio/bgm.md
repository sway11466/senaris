# BGM（音楽制作・組込）

BGM の制作方針と組込の運用。**費用をかけない**前提で、限られた戦力を盤面で噛み合わせて勝つ本作らしく、少ない曲を作り込んで回す。翻訳アセットの運用（[../tech/i18n.md](../tech/i18n.md)）と同じく「素材パイプラインの取り決め」を扱う。

## 前提

- **予算ゼロ**。有料の音源・外注は使わない（無料の範囲で完結させる）。
- 制作者はゼロから作曲はしないが、**楽譜が読め、ベースのメロディがあれば編曲・修整ができる**。
- 配布はまず Steam（PC）＝**商用**。使う素材はすべて商用利用可であることを確認する。

## 方針（二段構え）

曲を全部自作しようとしない。用途で手段を分ける。

- **通常曲（大半）**: 商用可のフリー BGM 素材を**そのまま使う**。選ぶだけ。
- **看板曲（数曲）**: タイトル／メインテーマ／ボス戦など、作品の顔になる曲だけ **MuseScore で自作アレンジ**する。
- 元手を増やさないため、**メインテーマの編成違い**（タイトル版・ボス版など）で流用し、統一感を出す。盤面同様「与えられた戦力を噛み合わせる」。

## ツール（すべて無料）

- **Muse Hub** — Muse 社の配布・更新ランチャー。下記を入れる入口。
- **MuseScore Studio**（旧 MuseScore 4）— 楽譜作成ソフト本体。制作・編集はこれ。楽譜が読める制作者に最も手が合う。
- **Muse Sounds** — 無料の高品質音源（オーケストラ等）。Muse Hub 経由で入れる。これで書いた曲がぐっと良く鳴る。
- 必要になったら無料 VST（Spitfire LABS / BBC SO Discover 等）＋無料 DAW（Cakewalk / Reaper / LMMS）に発展。ただし**まずは MuseScore 単体で完結**させる。

## データ形式

| 形式 | 拡張子 | 役割 | 扱い |
|---|---|---|---|
| MuseScore 原本 | `.mscz` | 編曲の原本 | **必ず残す**（後で直せる） |
| MusicXML | `.musicxml` | 記譜の交換フォーマット | ベースメロディの受け渡し・編集の入口 |
| MIDI | `.mid` | 演奏データ | フリー MIDI や生成物の取り込み・プレビュー |
| Ogg Vorbis | `.ogg` | 最終成果物 | Godot に載せる。原本から作り直せる生成物 |

- **原本 `.mscz` が正**。`.ogg` は書き出しなので git 管理は任意（生成物）。
- 他所からメロディをもらう時は **MusicXML か MIDI** で受け取り、MuseScore で開いて直す。MIDI→楽譜変換はリズム解釈がずれることがあるので取り込み後に手直し前提。

## 制作ワークフロー

```
入手（PD楽譜 .musicxml / フリー .mid）
  → 編曲（MuseScore、原本 .mscz で保存）
  → 書き出し（.ogg）
  → Godot 組込（AudioStream 登録・ループ設定）
```

## ベースメロディの入手（作曲しない前提の要）

商用の安全さを最優先し、**権利がクリーンな順**に使う。

1. **パブリックドメインのクラシック**（IMSLP 等）— 楽譜（作曲）が著作権切れ。**自分で写譜・アレンジすれば成果物は自作物**になり最も安全。※他者の「演奏録音」「現代の編曲版」は別権利なので、古い原譜から起こす。
2. **伝統曲・民謡（PD）** — ファンタジー舞台と相性が良い。旋律だけ借りて再編曲。
3. **商用可のフリー MIDI／モチーフ集** — ライセンスを確認して土台に。
4. **AI 生成は補助まで**（Suno 等）。商用ライセンスがプラン依存で、AI 生成物の著作権自体が各国で未確定。**看板曲ほど自分の改変を厚くし**、素性を自作寄りに寄せる。

> 補足: 生成ツールに旋律のたたき台（MusicXML/MIDI）を出させ、MuseScore で大幅に編曲する運用も可。あくまで「ベース」で、良し悪しの判断と仕上げは制作者が行う。

## 必要曲リスト（最小構成）

BGM は**ステージ単位**で流す。本作はマップそのものが戦場で1回の殴り合いが一瞬なので、**戦闘ごとに曲を切り替えず、攻撃・着弾は効果音（SFX）と戦闘演出で示す**（ネクタリス／Advance Wars 型。戦闘が別画面に遷移するファイアーエムブレム型とは分ける）。少数を作り込む。目安は 6〜8 曲。各曲に「狙う雰囲気・楽器・尺・ループ要否」を持たせる。

| 用途 | 雰囲気（狙い） | 主楽器の目安 | ループ |
|---|---|---|---|
| タイトル | 作品の顔。メインテーマ | オーケストラ | ○ |
| ステージ（通常） | おおらか・爽やか・広がり（FF系の悠久感） | フルート／弦 | ○ |
| ステージ（緊張・ボス） | 威圧・盛り上がり（メインテーマの編成違いで可） | 金管＋弦／打楽器 | ○ |
| 危機（状態切替） | 通常曲の暗転版（下記アダプティブ用） | 短調・低音・打楽器増 | ○ |
| 勝利 | 短い高揚 | 全体 | × |
| 敗北 | 短い沈み | 弦／独奏 | × |

> 「戦闘曲」を独立トラックとしては持たない。緊張感は**ステージ曲の出し分け＋下記の状態切り替え**で作る。個々の戦闘は SFX（[../tech/combat_scene.md](../tech/combat_scene.md) の演出）で示す。

## Godot 組込

- 形式は **`.ogg`**（軽量・ループ設定しやすい）。`AudioStreamOggVorbis` で読み、`AudioStreamPlayer` で再生。
- ループは Ogg 側 or インポート設定の **`loop` / `loop_offset`** で管理。ステージ中ずっと流れる曲はループ必須。
- レイヤー上、音の再生制御は presentation 寄り。曲の選択ルール（場面→曲）が要るなら application に薄く持たせ、`domain`/`data` は音に依存させない（[../tech/architecture.md](../tech/architecture.md)）。

## ループ設計の注意

- マップ曲などは延々流れるので、**終端が頭に自然につながる**よう末尾を整える。無音や余韻の尻切れに注意。
- MuseScore で書く段階から「ループ点」を意識し、最後の小節から 1 小節目への接続を確認する。

## 状態でBGMを切り替える（アダプティブ）

特定の状況で曲を切り替えられる。例: **敵が必殺技を使ったら危機BGMに変わり、以後クリア／敗北まで戻らない**。

- **方式**: まずは**2曲をクロスフェード**する単純切り替えで十分（1〜2秒）。音楽的にきっちり繋ぎたければ Godot 4 の **`AudioStreamInteractive`**（クリップ＋遷移ルール。次の小節頭で切替）を使う。レイヤー加算（`AudioStreamSynchronized`）は素材コストが高いので後回し。
- **「永続」＝状態フラグ1個**: ステージ状態に `crisis: bool` を持ち、一度立てたら戻さない（曲がパタパタ切り替わる事故を防ぐ）。ステージ開始でリセット。
- **責務分担**（[../tech/architecture.md](../tech/architecture.md)）:
  - `domain` … 「敵が必殺技を使った」という**事実をシグナル発火**するだけ（音を知らない）。
  - `application` … それを受けて BGM 状態を決める（`crisis` を1回だけ立てる）。
  - `presentation` … `AudioStreamPlayer` の差し替え／クロスフェードを実行。
- **危機BGMの用意**: 新曲は不要。**通常曲を短調化／テンポ上げ／低音・打楽器増**した“同テーマの編成違い”を同じ `.mscz` から派生（元手ゼロ）。ループ点整備は両曲に必要。
- **セーブ**: 途中セーブがあるなら `crisis` フラグも保存する（ロード時に通常曲へ戻る事故を防ぐ。[../tech/gamesystem.md](../tech/gamesystem.md)）。
- **引き金の粒度**: 切替は“盤面が変わる級”のイベント（必殺技・ボス出現など）に絞る。弱いイベントで変えると唐突。

## 権利・ライセンス台帳

商用（Steam）前提。使った素材は**必ず台帳に記録**する。有料データ保護の方針（[../adr/ADR-0002-paid-data-protection.md](../adr/ADR-0002-paid-data-protection.md)）とは別に、**こちらは「他者の権利を侵さない」ための管理**。

記録項目（曲ごと）:

- 曲名（本作での用途）／出典（サイト・IMSLP番号・PD原譜 等）
- 入手形式（PD楽譜／フリーMIDI／自作 等）
- ライセンス（商用可否・改変可否・**クレジット表記の要否**）
- 改変度（そのまま／編曲／自作）

## クレジット表記

- 表記が必要な素材は、**Steam ページとゲーム内クレジット**にまとめて記載する。
- 表記文はライセンスの指定文をそのまま使う（自己流に変えない）。
