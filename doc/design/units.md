# ユニット

ユニットの性能設計・陣営方針と、具体ステータス（対応表）。
名前・見た目はオリジナル、性能特性はネクタリスを下敷きにする。

> 関連: 戦闘解決は [combat.md](combat.md)／占領・拠点運用は [map.md](map.md)。

---

## 性能設計

- 名前・見た目はオリジナルのファンタジー風。
- 性能的な特性（移動力・攻撃／防御傾向・地形適性・経験値による成長など）はネクタリスを下敷きにする。
- どのネクタリス的特性を、どのファンタジーユニットに割り当てるかは未決（下記対応表）。
- **完全互換は目指さない**：ネクタリスの8カテゴリ23種をすべて引き継ぐ必要はない。役割は取捨選択してよい。機雷（＝ネクタリスのヤマアラシ）は**不採用**。【確定】
- **両刀（対地対空）ユニットの扱い**：対地・対空の両対応ユニットは、火力・機動・耐久のいずれかで明確に妥協させる（万能で器用貧乏にする）。【指針】
- **パラディン**：万能ゆえに強力なので、**厳選マップ限定**で運用する。【確定】

## 陣営方針：基本ミラー＋例外

基本方針は **「基本ミラー＋例外」**。

1. **敵味方はミラーを土台にする**：性能・役割は敵味方で同じ（名前と見た目だけ陣営別）。【確定】
   - → 役割＋ステータスの表は **1セット作れば両陣営に使える**。
2. **見た目だけ陣営別**：同じ役割でも、名前・グラフィックは陣営ごとに別物にする。【確定】
3. **片陣営専用の強ユニットを少数置いてよい（ハンター枠／例：ドラゴン）**：これがミラーに対する「例外」。
4. **専用強ユニット（ドラゴン）は占領で奪えない（暫定 / ※要確定）**：敵専用の強ユニットは占領しても自軍に編入しない。【暫定】
   - 特別感を保てる。
   - 拠点占領による増援との衝突も回避できる（→ [map.md](map.md)）。

## ユニット対応表（テーマ0／標準）

ネクタリスの性能を引き継いだユニット表。**敵味方はステータス・役割がミラー**で、名前とグラフィックのみ陣営別。
この表の「ステータス」が**共有テンプレート**で、「味方／敵」はその最初のスキン。
フィールドのテーマ（古代遺跡・死霊の洞窟など）ごとに、**同ステータス・別名のエネミー列を増やす**運用（＝この表はテーマ0／標準。テーマ別リスキン方式は [map.md](map.md) 参照）。

### 補足メモ

- **ドラゴン（敵専用）**: ネクタリスのハンター枠＝万能の強ユニット。味方版なし。占領しても奪えない（暫定）。
- **パラディン**: 万能エリートのため**厳選マップ限定**で運用（万能ユニットはバランスを壊しやすい＝ハンターの教訓）。
- **占領の3層**: クレリック（安価・脆い）／ビショップ（高耐久・保持）／パラディン（戦える・エリート）。→ 運用は [map.md](map.md)。
- **両刀（対地対空）ユニットの妥協**: アーチャー＝低火力、スナイパー＝脆い、ニンジャ＝脆い。パラディンのみ弱点が薄い（要・運用制限）。
- **不採用**: ヤマアラシ（機雷）。完全互換は目指さない方針。
